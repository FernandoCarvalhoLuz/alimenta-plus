O Garçom (Frontend)

Vue.js 3 + Vite

Fica no salão (navegador do usuário). É a cara do sistema. Ele recolhe as informações preenchidas pelo Doador ou pela ONG, busca automaticamente o endereço ao digitar o CEP e cria o modal interativo dos termos jurídicos.

// Coleta dados da tela const form = ref({ nome: 'Ricardo Doador', documento: '123.456.789-00', aceitou_termo_juridico: true });
🧑‍🍳

O Chef (Backend)

Node.js + Express + Mongoose

Fica na cozinha reservada (servidor). Quando o garçom traz o cadastro, ele inspeciona rigorosamente: "Esse usuário aceitou as obrigações legais da doação?". Se a validação passar, ele manda para o banco de dados.

// Valida termos jurídicos if (!aceitou_termo_juridico) { return res.status(400).json({ error: "Obrigatorio!" }); }
🗄️

A Despensa (Banco de Dados)

MongoDB Atlas Cloud

É a dispensa organizada e segura (banco de dados na nuvem). Guarda os registros de doadores, ONGs, doações e termos aceitos de forma estruturada para que possam ser resgatados a qualquer instante.

// Schema do usuário no Mongoose const usuarioSchema = new mongoose.Schema({ nome: String, documento: String, aceitou_termo_juridico: Boolean });
Etapa 1

Definição do Modelo do Banco (Schema)

Modelamos o esquema Usuario.js com Mongoose, definindo que campos como perfil (DOADOR ou ONG) e aceitação jurídica são de gravação obrigatória.

Etapa 2

Construção da Tela em Vue.js

Criação do componente de cadastro com estilos em Glassmorphism, integração com ViaCEP (para autocompletar rua/bairro) e gerenciamento reativo do formulário.

Etapa 3

Implementação do Modal Jurídico

Desenvolvimento do termo legal com base na Lei brasileira de doação de alimentos (Lei nº 14.016/2020) no formato popup, isolando o clique do link do controle do checkbox.

Etapa 4

Desenvolvimento das Rotas e Conectividade

Configuração do servidor Express em server.js, criação de variáveis de ambiente seguras (.env) e roteamento de requisições POST para gravação de usuários no MongoDB Atlas.

Etapa 5

Testes de Integração e Feedback

Criação de tratamento de erros de DNS no Node.js para redes complexas e ativação do feedback por alertas visuais de sucesso ou falha, limpando o formulário após a gravação.

1
Criar o Modelo de Usuário (O Schema)
backend/src/models/Usuario.js

O primeiro passo de qualquer funcionalidade no backend é definir a estrutura do dado que vamos guardar no banco de dados. Com o Mongoose, criamos um Schema (um molde ou modelo) que força os registros a seguirem regras estritas de tipos de dados e campos obrigatórios.

💡 O que esse código faz? Ele diz ao MongoDB: "Só aceite cadastrar o usuário se ele me der o Nome (String), o tipo de Perfil (Doador ou ONG), o Nome da Empresa (String) e se ele aceitou expressamente o checkbox do Termo Jurídico (Boolean)".
Código Criado:
const mongoose = require('mongoose'); const usuarioSchema = new mongoose.Schema({ nome: { type: String, required: true }, tipo_perfil: { type: String, enum: ['DOADOR', 'ONG'], required: true }, empresa: { type: String, required: true }, documento: { type: String, required: true }, aceitou_termo_juridico: { type: Boolean, required: true }, reputacao: { type: Number, default: 5 }, endereco: { cep: String, bairro: String, logradouro: String, numero: String, complemento: String }, data_cadastro: { type: Date, default: Date.now } }); module.exports = mongoose.model('Usuario', usuarioSchema);
2
Criar a Rota de Cadastro (POST)
backend/src/routes/usuarios.js

A rota é o "canal de escuta" que criamos no nosso servidor para receber chamadas de rede. Usamos o verbo POST para indicar que essa chamada criará um novo registro no banco de dados.

🔒 Regra de Negócio Crucial (Onda 1): Veja que na linha 14 nós barramos a requisição no servidor se o usuário tentar burlar a tela e enviar sem o aceitou_termo_juridico marcado. Segurança jurídica em primeiro lugar!
Código Criado:
const express = require('express'); const router = express.Router(); const Usuario = require('../models/Usuario'); router.post('/', async (req, res) => { try { const { nome, tipo_perfil, empresa, documento, aceitou_termo_juridico, endereco } = req.body; if (!documento) { return res.status(400).json({ error: "O CPF ou CNPJ (documento) é obrigatório." }); } // VALIDAÇÃO JURÍDICA JÁ NO SERVIDOR if (!aceitou_termo_juridico) { return res.status(400).json({ error: "É obrigatório aceitar o Termo de Segurança Jurídica." }); } const novoUsuario = new Usuario({ nome, tipo_perfil, empresa, documento, aceitou_termo_juridico, endereco }); await novoUsuario.save(); // Salva fisicamente no Atlas res.status(201).json({ message: "Usuário cadastrado com sucesso!", usuario: novoUsuario }); } catch (error) { res.status(500).json({ error: "Erro interno ao cadastrar.", detalhes: error.message }); } }); module.exports = router;
3
Conectar tudo e configurar o DNS
backend/server.js

O arquivo server.js é a engrenagem principal que lê as variáveis de ambiente, inicializa as rotas que criamos acima e inicia a conexão com o MongoDB Atlas usando a biblioteca Mongoose.

⚡ O Ajuste Secreto de DNS: Para resolver o erro querySrv ECONNREFUSED que acontece em redes domésticas e universitárias, importamos o módulo dns nativo do Node e forçamos ele a priorizar o IPv4 e os servidores DNS do Google (8.8.8.8) e Cloudflare (1.1.1.1).
Código Criado:
require('dotenv').config(); // Ajuste para evitar erros de DNS no Atlas const dns = require('dns'); dns.setDefaultResultOrder('ipv4first'); try { dns.setServers(['8.8.8.8', '1.1.1.1']); } catch(e) {} const express = require('express'); const cors = require('cors'); const mongoose = require('mongoose'); const app = express(); app.use(cors()); app.use(express.json()); // Conexão com Atlas mongoose.connect(process.env.MONGO_URI) .then(() => console.log('Conexão estabelecida com sucesso!')) .catch((err) => console.error('Erro ao conectar:', err.message)); // Importa e registra as rotas de usuários const usuariosRouter = require('./src/routes/usuarios'); app.use('/api/usuarios', usuariosRouter); app.listen(3000, () => console.log('Servidor rodando na porta 3000'));
4
Criar a Interface com Autopreenchimento de CEP e Modal
frontend/src/components/CadastroUsuario.vue

No Frontend criamos a tela de cadastro em Vue 3 usando composição reativa (ref). Ela gerencia o preenchimento dos campos, o disparo de API para buscar os CEPs automaticamente, o modal flutuante com a Lei nº 14.016/2020 e a submissão dos dados.

🌟 Validação e Feedback Amigável: Observe que a função cadastrar agora aguarda a resposta do backend e exibe um alert() nativo e limpa o formulário apenas se der tudo certo (código HTTP 201), dando feedback real para o usuário!
Trecho da Lógica em Javascript:
const form = ref({ nome: '', tipo_perfil: 'DOADOR', empresa: '', documento: '', aceitou_termo_juridico: false, endereco: { cep: '', bairro: '', logradouro: '', numero: '', complemento: '' } }); const exibirModalTermos = ref(false); // Busca o CEP na API ViaCEP do governo const buscarCep = async () => { const cep = form.value.endereco.cep.replace(/\D/g, ''); if (cep.length !== 8) return; try { const response = await fetch(`https://viacep.com.br/ws/${cep}/json/`); const data = await response.json(); if (!data.erro) { form.value.endereco.bairro = data.bairro; form.value.endereco.logradouro = data.logradouro; } } catch(e) {} }; // Dispara o cadastro para o Express const cadastrar = async () => { try { const response = await fetch('http://localhost:3000/api/usuarios', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(form.value) }); const data = await response.json(); if (response.ok) { alert("Conta criada com sucesso! " + data.message); // Limpa os dados do formulário form.value = { nome: '', tipo_perfil: 'DOADOR', empresa: '', documento: '', aceitou_termo_juridico: false, endereco: { cep: '', bairro: '', logradouro: '', numero: '', complemento: '' } }; } else { alert("Erro ao cadastrar: " + data.error); } } catch(e) { alert("Erro de conexão!"); } };
