1. O Problema e a Solução (Inception)

Através do Lean Inception, criamos Personas para entender quem utilizará o sistema. Nosso MVP é focado na Onda 1 e 2.

👤 Doador (Ricardo)

Precisa de um sistema ágil (Cadastro Expresso) no final do expediente para não perder tempo jogando comida fora e precisa de respaldo Jurídico.

🤝 Receptora (ONG - Sandra)

Precisa de um painel confiável onde ela possa ver as doações e Reservar os itens para evitar viagens perdidas.

2. A Arquitetura Tecnológica (MEVN Stack)

Escolhemos a stack moderna baseada inteiramente em JavaScript para unificar a linguagem de ponta a ponta.

M - MongoDB

Banco de dados NoSQL Orientado a Documentos. Hospedado na nuvem (Atlas) para agilidade e escalabilidade sem precisar instalar nada na faculdade.

E & N - Express + Node.js

O cérebro do projeto. Uma API RESTful que gerencia as reservas e as regras de negócio antes de salvar no banco.

V - Vue.js

Interface SPA (Single Page Application) responsiva. Perfeito para rodar rápido no celular do Doador.

3. Estrutura do GitHub e Pastas

Dividimos o nosso repositório profissionalmente. Cada pasta tem um papel fundamental:

  • /backend/: Onde roda o Node.js. Já contendo o arquivo `package.json` (que dita quais bibliotecas vamos baixar) e o `server.js` (o motor da API).
  • /frontend/: Onde roda o Vue.js. Também contém seu próprio `package.json` para instalar as bibliotecas de tela.
  • /banco_de_dados/: Onde ficam nossos scripts `.js` de inicialização (Seed) para limpar e popular o banco automaticamente.
  • /docs/: Toda a inteligência do negócio (Este roteiro, o Canvas MVP e as Histórias de Usuário).

E na raiz do projeto temos o README.md, que é o mapa do tesouro ensinando qualquer desenvolvedor a rodar o app.

4. Subindo a Aplicação (O que fizemos na prática)

O professor pediu que a aplicação já pudesse ser "levantada". Explicando os comandos que usamos no VS Code:

  1. O Comando Mágico de Instalação: npm install Explicando para a turma: O Node Package Manager lê o nosso package.json e vai na internet buscar tudo o que precisamos (Vue, Express, Mongoose) salvando na pasta oculta node_modules.
  2. Ligando o Motor do Backend: npm run dev Explicando: Esse comando acorda o nosso arquivo server.js na porta 3000. O sufixo "dev" ativa o Nodemon, que reinicia o servidor sozinho a cada vez que a gente salva o código.
  3. Ligando a Vitrine do Frontend: npm run serve Explicando: Inicia a tela do usuário na porta 8080 usando o Vite, que compila a tela na velocidade da luz!

5. O Banco de Dados e a Nuvem (Demonstração)

Nós modelamos a coleção de Usuários e Doações através do Mongoose. Em vez de rodar local, levamos para a Nuvem!

O Script de Seed

Nós usamos o terminal conectado à nuvem para rodar a função load():

load("banco_de_dados/seed.js")

Isso inseriu os dados de teste no MongoDB Atlas instantaneamente.

*Dica para a apresentação: Neste momento, abra o site do MongoDB Atlas ou o MongoPlayground na tela e mostre os dados em JSON (Coleção usuarios e doacoes) comprovando a conexão.*

6. Versionamento (Como trabalhamos em equipe)

Para garantir que ninguém perca o trabalho do outro, usamos o Triângulo de Ouro do Git no final de cada dia:

1. git add .

Prepara todos os arquivos modificados (Coloca no carrinho).

2. git commit -m "mensagem"

Bate a foto do código e coloca uma etiqueta explicando o que foi feito.

3. git push

Envia a atualização com segurança lá para o nosso repositório no GitHub para todos poderem baixar com o git pull!
