💡 Sobre este Modelo Lógico para o Professor:

Embora o nosso projeto utilize MongoDB (NoSQL), é super comum na academia pedirmos o modelo relacional (ER) para comprovar que a equipe entende a estrutura lógica dos dados antes de implementá-los.

Neste diagrama em Pé de Galinha, nós mostramos as seguintes Cardinalidades:

  • 1 para N (1:N) - PUBLICA: 1 Usuário (do tipo Doador) pode publicar 0 ou N Doações. O campo doador_id na Doação é a Chave Estrangeira (FK).
  • 1 para N (1:N) - RESERVA: 1 Usuário (do tipo ONG) pode reservar 0 ou N Doações. O campo ong_reservou_id é a FK (podendo ser nula se a doação ainda estiver DISPONIVEL).
