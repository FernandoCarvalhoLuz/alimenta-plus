💡 Sobre este Modelo Lógico:

Embora o nosso projeto utilize MongoDB (NoSQL), este modelo relacional ilustra a estrutura física utilizando uma tabela de junção para a transação de reservas, conectando as partes por ponteiros (Chaves Estrangeiras).

Neste diagrama em Pé de Galinha, nós mostramos as seguintes Cardinalidades:

  • USUARIO para DOACAO (1:N): 1 Usuário (Doador) publica 0 ou N Doações. O campo doador_id aponta para o ID do usuário correspondente.
  • RESERVA (Tabela de Junção): A entidade RESERVA vincula uma transação contendo os ponteiros ong_id (aponta para quem está reservando na tabela USUARIO) e doacao_id (aponta para o item reservado na tabela DOACAO).
